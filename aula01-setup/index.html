Preparando seu Ambiente (Setup)

Nesta aula você vai preparar o ambiente de código. Vamos instalar as extensões e ajustar a máquina para rodar nossos scripts HTML e Tailwind locais.

Ao final desta aula você será capaz de:

  • Instalar e abrir o VS Code no Windows.
  • Configurar as extensões (Live Server, VS Code Speech).
  • Acessar o repositório oficial de skills.
  • Estruturar a pasta inicial do curso.

Contexto de Governo:

  • Uso otimizado sem permissões avançadas (zero npm).
  • Dicas para contornar bloqueios de proxy/firewall.
  • Integrações n8n baseadas em segurança.

Passo a passo: Instalação das Ferramentas

Siga os passos abaixo na sua máquina e marque o checklist.

1. Instalar o VS Code

  1. Acesse: code.visualstudio.com/Download.
  2. Baixe o instalador User Installer para Windows.
  3. Execute e mantenha as marcações padrão.

2. Instalar Extensões

Pressione Ctrl+Shift+X e instale:

  • Live Server – simula um servidor local instantâneo.
  • Tailwind CSS IntelliSense – digitação rápida de classes.
  • VS Code Speech (Microsoft) – possibilita programar emitindo comandos com a sua voz pelo microfone.
  • Portuguese Language Pack – traduz a ferramenta.

Dica TI (Se bloqueado)

Caso o download não funcione, peça abertura de chamado à TI. Diga que usará o VS Code com tecnologias nativas do navegador em prol do projeto Vibe Coding governamental.

Instruções rápidas: instalar o Antigravity (opções) e criar conta no GitHub.

Instalar o Antigriver (IDE do Google)

O Antigriver é a IDE oficial do Google para desenvolvimento (desktop e/ou versão web). Use o download oficial abaixo ou prefira a versão web quando sua máquina tiver restrições.

  • Baixe o instalador oficial: https://antigravity.google/download. Execute o instalador no Windows como Administrador.
  • Versão web (se disponível): abra o mesmo link no navegador e use a IDE sem instalar — útil em máquinas restritas.
  • Instalação via gerenciador (quando disponível): verifique o nome do pacote e rode winget install Google.Antigriver ou choco install antigriver. Confirme o identificador do pacote antes de executar.
Dica: se sua máquina estiver bloqueada pela TI, peça ao time de infraestrutura para instalar ou use a versão web quando possível.

Criar uma conta no GitHub

  1. Acesse github.com/join.
  2. Preencha nome de usuário, e-mail e senha. Use um e-mail institucional se preferir separar contas.
  3. Verifique seu e-mail (receberá um link de confirmação).
  4. Configure a autenticação em dois fatores (2FA) para segurança recomendada pelo time de TI.
  5. Instale o Git localmente (se ainda não): git --version para checar, ou baixe em git-scm.com.

Para que serve o GitHub

Resumo prático:

  • Controle de versão: acompanhar alterações no código e voltar a versões anteriores.
  • Colaboração: contribuir com colegas, revisar código e gerenciar tarefas via Pull Requests e Issues.
  • Backup e sincronização: repositórios remotos servem como cópia segura e histórica do trabalho.
  • Automação: GitHub Actions roda testes e deploys automaticamente.
  • Documentação e sites: GitHub Pages publica páginas estáticas (útil para materiais e portfólios).
